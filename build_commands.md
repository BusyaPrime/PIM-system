# OmniPIM-MAX Build Commands
Вот точные команды с учетом перехода в директории:

**Диагностика упавшего бэкенда (порт 8000):**
Выполни эти команды по очереди и скинь мне их вывод — это 100% решит проблему!

```powershell
# 1. Проверяем, жив ли контейнер php вообще
docker compose ps php

# 2. Смотрим последние логи ошибок (почему он упал)
docker compose logs --tail=50 php

# 3. Принудительно стартуем сервер, если он завис
docker compose restart php
```
